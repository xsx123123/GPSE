# Change Log

## 2026-07-18 — Canonical SNP IDs and Prediction Alignment

### Summary

Implemented a shared SNP identity contract across `convert`, `train`, and `predict` so that trained GPSE models can be reused with VCF files from other users and projects.

### Canonical SNP ID

All SNPs generated by the PLINK/PED/MAP-to-matrix conversion path now use:

```text
chr<chrom>_<chromStart>_<chromEnd>
```

The coordinate convention is UCSC-style zero-based, half-open coordinates. For example:

```text
VCF chr1:100, REF=A  ->  chr1_99_100
VCF chrX:7, REF=AT  ->  chrX_6_8
```

The shared implementation is in `gpse/utils/snp_ids.py`.

### Convert Changes

- `gpse.convert.genotype_matrix.convert_to_matrix()` now derives SNP IDs from MAP/BIM chromosome and base-pair coordinates instead of retaining arbitrary PLINK SNP names.
- Converted matrices write an adjacent feature manifest:

  ```text
  <matrix-stem>.features.json
  ```

- Existing matrices without a feature manifest are regenerated instead of being silently reused with potentially outdated SNP names.
- Duplicate or malformed canonical SNP IDs now fail early with a clear error.
- User-provided `--matrix-file` inputs retain their existing column names because genomic coordinates cannot be reconstructed from arbitrary column names. Such manifests are marked as `custom`.

### Train Changes

- Training no longer renames genotype columns to `feature_0`, `feature_1`, etc.
- The exact ordered feature list used during training is saved to:

  ```text
  <results_dir>/feature_manifest.json
  ```

- Model scalers and estimators therefore retain the real SNP feature names and order.
- Legacy matrices with custom feature names remain supported, but their manifests are marked as `custom`.
- CV fold assignment now uses a dedicated reproducibility seed of `42` through `random.Random(42)`, so regenerating CV files with the same input produces identical folds regardless of global random state.
- Also fixed several training robustness issues:
  - non-shared test-set execution no longer loses the generated test indices;
  - multi-class fold ensemble uses averaged probabilities/majority voting instead of averaging class numbers;
  - Optuna early stopping now calls `study.stop()` with the correct no-improvement condition;
  - classification `all_predictions.json` is now populated like regression outputs.
  - Optuna and final CV evaluation now create a fresh estimator for every fold to prevent state carry-over.

### Predict Changes

`gpse predict` is now implemented for VCF and converted genotype matrices.

Example:

```bash
gpse predict \
    --model results/ \
    --vcf-file new_samples.vcf.gz \
    --out predictions.csv
```

The workflow:

1. Converts VCF `CHROM/POS/REF` records to canonical SNP IDs.
2. Loads the model's ordered `feature_manifest.json`.
3. Reorders input features to exactly match training order.
4. Reports model SNPs missing from the input.
5. Reports extra input SNPs not used by the model.
6. Fills absent model SNPs with GPSE's missing genotype code `3` by default.
7. Writes predictions and a JSON alignment report.

Generated files:

```text
predictions.csv
predictions.alignment.json
```

The missing-value behavior can be changed with:

```bash
--missing-value VALUE
```

A model without a feature manifest is rejected when safe feature alignment cannot be established.

### Tests and Validation

- Added `tests/test_snp_id_alignment.py` covering canonical ID generation, convert output, manifests, and missing-feature prediction alignment.
- Verified regression prediction, three-class prediction, VCF prediction, missing/extra SNP reporting, and top-level CLI routing.
- Full test suite result: `16 passed`.
